# Frontend Review Skill

Use this skill when reviewing frontend code.

## Checklist

1. Structure
   - Components are small.
   - Pages are separated from reusable components.
   - API logic is centralized.
   - State management is not overcomplicated.

2. API Integration
   - Loading states exist.
   - Error states exist.
   - Empty states exist.
   - Duplicate requests are avoided.
   - Auth failures are handled.

3. Forms
   - Client-side validation exists.
   - Server-side validation errors are displayed.
   - Duplicate submit is prevented.
   - Inputs have labels.

4. UX
   - Mobile-first layout.
   - Touch targets are large enough.
   - Navigation is clear.
   - Dark mode does not break contrast.

5. Accessibility
   - Semantic HTML.
   - Keyboard navigation.
   - Proper labels.
   - Meaningful alt text.
   - No critical action hidden behind hover only.

## Output Format

Return:

1. UX quality
2. Code quality
3. Bugs or edge cases
4. Accessibility issues
5. Suggested improvements
