# Sprint Planning Skill

Use this skill when planning EDUCYCLE work.

## Planning Rules

- Separate backend and frontend work.
- Identify dependencies between tasks.
- Prioritize security and core flows first.
- Avoid vague tasks like "improve UI" without acceptance criteria.
- Each task must have a clear output.
- Each task should be small enough to review.

## Task Format

Each task should include:

- Title
- Type: Backend / Frontend / DevOps / Security / Testing / Docs
- Priority: P0 / P1 / P2 / P3
- Description
- Acceptance criteria
- Files or modules likely affected
- Test requirement

## Sprint Output Format

Return:

1. Sprint goal
2. Backend tasks
3. Frontend tasks
4. Security tasks
5. Testing tasks
6. Definition of done
7. Risks
